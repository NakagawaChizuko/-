# 化石・遺物入力アプリ（スマホ/タブレット対応）

## できること
- 統合入力フォームで重複項目を1回だけ入力
- 入力データを「リスト形式」「カード形式」でプレビュー
- リストCSV・カードCSV・JSONを書き出し
- JSONを再読み込みして復元
- 写真を端末カメラまたはファイルから添付
- Google Apps Script経由で共有Googleドライブへ保存（任意設定）

## ファイル
- `index.html`: 画面
- `styles.css`: 見た目
- `app.js`: 入力・保存ロジック
- `google_apps_script.gs`: 共有Googleドライブ保存用API（Apps Script）

## 使い方（同一Wi-Fiで入力する場合）
1. このフォルダを開いて、次を実行します。  
   `python3 -m http.server 8080`
2. PCブラウザで確認:  
   `http://localhost:8080/kaseki_mobile_app/`
3. スマホ/タブレットは、同じWi-Fi上でPCのIPアドレスを使ってアクセスします。  
   例: `http://192.168.1.10:8080/kaseki_mobile_app/`

## 共有Googleドライブ保存の設定
1. `google_apps_script.gs` の内容をGoogle Apps Scriptに貼り付けてデプロイします。
2. ウェブアプリURLをコピーします。
3. 本アプリ右上の `Google Apps Script WebアプリURL` に貼り付けます。
4. `共有保存を設定` を押します。
5. 表示が `保存先: 共有Googleドライブ` になれば有効です。

補足:
- `クラウド再読込` で最新データを手動取得できます。
- `共有保存OFF` で端末保存のみ（localStorage）に戻せます。
- 同時編集の取りこぼしを減らすため、`google_apps_script.gs` は最新内容へ貼り替えて再デプロイしてください（保存時にロック + レコード単位マージ）。

## 入力ルール（現在設定）
- 区画は `24-Ⅰ-C-4` 形式（4要素すべて編集可）
- 標本番号は `m-1` 形式（分類タブ + 数字入力）
- 分類タブ: `m: 哺乳類 / a: 分析用資料 / b: 植物 / l: 生痕 / s: 貝類 / i: 昆虫 / g: 人類考古 / h: その他`
- `a: 分析用資料` を選んだ場合は区分選択:
  - `A: 火山灰 / C: 14C / M: 古地磁気 / F: フィッショントラック / P: 花粉 / B: 植物 / I: 昆虫 / D: 珪藻 / R: 粒度 / S: 貝類 / H: その他 / Mg: はぎとり資料`
- リスト側の主要項目: `レベル読値（上面cm・下底cm） / 産出状況断面(要・否) / 産状スケッチ(要・否)`
- 平面位置はタブ選択（`北から / 南から`、`東から / 西から`）+ 数値入力
- 地層名はタブ選択:
  - `1.芙蓉湖砂シルト部層`
  - `2.立が鼻砂部層`
  - `3.海端砂シルト部層`
  - `4.その他`（内容を自由入力）

## 注意点
- デフォルトの保存先はブラウザの`localStorage`です（同じ端末・同じブラウザ内のみ）。
- 写真が多すぎると保存上限に達する場合があります。必要に応じてJSON書き出しを行ってください。
- CSVはExcelで開けます（写真本体は含まず、枚数のみ出力）。
- 写真は、クリノメーター（方位の分かるもの）と定規を一緒に写して記録してください。
- 共有Googleドライブ保存を有効化した場合は、クラウド保存と端末保存の両方を行います。
